ChickenBus

The primary functions of this application are:

  • For travelers or locals to get information about bus routes (and eventually all forms of public transportation) in lesser developed countries.
  • For employees/administrators of ChickenBus to be able to add information about bus routes through the website/mobile app directly to the database.

Requirements (prioritized):

  • Ensure the correct bus route to reach destination is returned
  • Simple, easy to use interface
  • Well structured and flexible database
  • Acurately provide critical travel information (bus time table, distance) that's easy to understand
  • A relatively efficient algorithm for finding the appropriate bus routes based on the users input
  • Ability for users with appropriate privileges to add information about bus routes using the web/mobile app
  • Cross platform compatability (in case Eric needs to switch off of Carolina Cloud Apps at a later date)
  • Ability for users to plan a trip out and save it to an itinerary for later use/reference
  • Ensure application supports multiple languages (especially Spanish)

User Types

  • Tourist/Budget Traveler
    • Privilege: Control their logins
  • Local
    • Privilege: Control their logins and the ability to verify/confirm route info
  • Employee
    • Privilege: Control their logins and add route info from the application. The added routes will updated in the database and be reflected in the application

Use Cases

  • Tourist/Budget Traveler
    1. To look up bus routes in lesser developed countries
        Input origin and destination locations and get a list of bus routes
  • Local
  • Employee
    1. Login with employee credentials to access admin functions
    2. An additional tab is accessible only with empolyee account
      1. The new tab will appear as a third option in the upper left corner of the application
    3. New tab will consist of a clickable map with text fields for entering bus stops attributes
    4. Add new bus stops/routes
      1. Employee can drop pins on the map to indicate the location of the new bus stop. The address will be generated
        • Alternatively, the pin can be generated by entering the address of the stop
      2. Additionally, multiple pins can be dropped to indicate a new route
    5. Edit attributes for new stops/routes
      1. By clicking on any pin, that stop's attributes will appear on the left side of the application
        • Similarly, attributes for the route associated with that stop will also be displayed
      2. Some fields such as direction and the stop number in a route are required, while others are optional
      3. The employee will fill out the relevent information for each stop and submit
      4. Following the submission, the information will be validated and added to the database
