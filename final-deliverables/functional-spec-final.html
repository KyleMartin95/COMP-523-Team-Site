ChickenBus

The primary functions of this application are:

  • For travelers or locals to get information about bus routes (and eventually all forms of public transportation) in lesser developed countries.
  • For employees/administrators of ChickenBus to be able to add information about bus routes through the website directly to the database.

Requirements (prioritized):

  • Web interface, ✓
  • Route entry, ✓
  • Mobile interface, ✓
  • Database web access page, ✓
  • Login/Registration, ✓
  • Bulk import, ✓
  • Bus schedule, ✓
  • Cross platform compatability, ✓

User Types

  • Tourist/Budget Traveler
    • Privilege: Control their logins
    • Access to search page and mobile app
  • Local
    • Privilege: Control their logins, (future) The ability to verify/confirm route info
    • Access to search page and mobile app
  • Employee
    • Privilege: Control their logins and add route info from the application. The added routes will updated in the database and be reflected in the application
    • Access to search page, route entry page, bulk entry, database visualization, and mobile app

Use Cases

  • Tourist/Budget Traveler
    1. To look up bus routes in lesser developed countries
        Input origin and destination locations and get the best bus route and any relevent information about those bus routes
  • Local
    1. To look up bus routes in their city/town if they need info on a route they are not already familiar with
  • Employee
    1. Login with employee credentials to access admin functions
    2. An additional tab is accessible only with empolyee account
      1. The new tab will appear as a third option in the upper left corner of the application
    3. New tab will consist of a map with text fields for entering bus stops attributes
    4. Add new bus stops/routes
        • The pin can be generated by entering the address of the stop
    5. Edit attributes for new stops/routes
        • Attributes for the route associated with that stop will be displayed on the left side of the application
      1. Some fields such as direction and the stop number in a route are required, while others are optional
      2. The employee will fill out the relevent information for each stop and submit
      3. Following the submission, the information will be validated and added to the database
