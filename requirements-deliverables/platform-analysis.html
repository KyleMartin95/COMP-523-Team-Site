Platform Analysis and Selection

We chose to use a combination of ReactJS as our front-end, ExpressJS and NodeJS for back-end, MySQL as our database, and React Native as our mobile solution.

Front-end: Choice for ReactJS was to allow better cross-platform adaptation when we eventually move to a mobile implementation with React Native
Back-end: Choice for NodeJS + ExpressJS was to have a uniform language across the application (JavaScript), as well as make maintenance easier
Database: Choice for MySQL was becasue we need a relational data base for routing purposes, also due to prior experiences
Mobile: Choice for React Native allows for direct transition from Web based appliaction to mobile without additional native development

Alternatives

Front-end

AngularJS
Pros: Structured MVC model, two way data binding
Cons: Less compaitible for mobile platforms than React

No framework
Pro: None
Cons: Slower development, no access to library functions

Back-end

Java or Ruby on Rails
Pros: More robust back-end functions with greater expandability
Cons: Different langauage, slower development cycle

Mobile

Cordova
Pros: Implmentable with just HTML, CSS, and Javascript alone
Cons: Less fluid scrolling on IOS, less close knit compatibility with ReactJS

Swift + Java
Pros: API access for all platform functionalities, more stable application
Cons: Need to develop on multiple platforms, long learning curve and development cycle unfit for project

Database

PostgreSQL
Pros: Object relational
Cons: Less experience

MongoDB
Pros: JSON like syntax for storing data, parse easily
Cons: not relational
